## 7. 安装 Redis

```bash
apt install -y redis-server
systemctl enable redis-server
systemctl start redis-server
redis-cli ping
```

限制 Redis 内存，避免单机被 Redis 吃满：

```bash
cp /etc/redis/redis.conf /etc/redis/redis.conf.bak.$(date +%Y%m%d%H%M%S)
sed -i 's/^supervised .*/supervised systemd/' /etc/redis/redis.conf
sed -i 's/^bind .*/bind 127.0.0.1 ::1/' /etc/redis/redis.conf
sed -i 's/^protected-mode .*/protected-mode yes/' /etc/redis/redis.conf
grep -q '^maxmemory ' /etc/redis/redis.conf && sed -i 's/^maxmemory .*/maxmemory 512mb/' /etc/redis/redis.conf || echo 'maxmemory 512mb' >> /etc/redis/redis.conf
grep -q '^maxmemory-policy ' /etc/redis/redis.conf && sed -i 's/^maxmemory-policy .*/maxmemory-policy allkeys-lru/' /etc/redis/redis.conf || echo 'maxmemory-policy allkeys-lru' >> /etc/redis/redis.conf
systemctl restart redis-server
redis-cli CONFIG GET maxmemory
```

确认 Redis 只监听本机：

```bash
ss -lntp | grep 6379 || true
```

## 8. 安装 Nginx

```bash
apt install -y nginx
systemctl enable nginx
systemctl start nginx
systemctl status nginx --no-pager
nginx -v
```

创建 DreamWeave 反代配置：

```bash
cat > /etc/nginx/sites-available/dreamweave <<'EOF'
server {
    listen 80;
    server_name 47.101.67.241;

    client_max_body_size 2m;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /ws/worker {
        proxy_pass http://127.0.0.1:3000/ws/worker;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 3600s;
        proxy_send_timeout 3600s;
    }
}
EOF

ln -sf /etc/nginx/sites-available/dreamweave /etc/nginx/sites-enabled/dreamweave
rm -f /etc/nginx/sites-enabled/default
nginx -t
systemctl reload nginx
```

验证：

```bash
curl -I http://47.101.67.241/
```

首次执行到这里时，如果 Node server 还没有部署，可能返回 `502 Bad Gateway`。这是正常的。

## 11.1 Python Worker 和向量库部署位置

DreamWeave 的 Python Worker 负责 AI 生成和 RAG 向量检索。当前默认架构是：

```text
云服务器：Node.js Server + PostgreSQL + Nginx + PM2
本地电脑：Python local-ai-worker + Ollama + Chroma / sentence-transformers
```

因此，Chroma、sentence-transformers 和 `apps/local-ai-worker/requirements.txt` 应安装在实际运行 Worker 的机器上。

如果 Worker 仍在本地 Windows 电脑运行，不要在云服务器上安装 Python 向量库，也不要在云服务器上启动 `dreamweave-worker`。云服务器只需要部署 Node.js Server、执行数据库 migration，并保持 `/ws/worker` 等待本地 Worker 主动连接。

## 11.2 本地电脑安装 Worker 向量库依赖

## 11.3 可选：把 Worker 也部署到云服务器

只有在你决定让云服务器同时运行 Python Worker 时，才执行本节。这样做通常要求云服务器也能访问 Ollama 或其他模型服务。

**重要**：由于 Ubuntu 24.04 + Python 3.12 有 PEP 668 保护，推荐使用虚拟环境安装依赖。

安装 Python 依赖（使用虚拟环境）：

```bash
cd /opt/dreamweave

# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 在虚拟环境中安装依赖
python -m pip install --upgrade pip
python -m pip install "chromadb>=0.4.0" "sentence-transformers>=2.2.0"

# 或者安装 requirements.txt 中的所有依赖
python -m pip install -r current/apps/local-ai-worker/requirements.txt

# 验证关键依赖
python -c "import chromadb; import sentence_transformers; print('RAG dependencies OK')"

# 退出虚拟环境（暂时不退出，后续步骤需要）
deactivate
```

如果遇到 "externally-managed-environment" 错误，说明系统启用了 Python 环境保护，虚拟环境是推荐的解决方案。

### 11.3.2 云服务器 Chroma 向量库说明

如果 Worker 运行在云服务器，Chroma 向量库会自动在 Worker 启动目录下创建 `chroma_db` 文件夹：

- 存储路径：`/opt/dreamweave/current/chroma_db`
- 每个故事的 Lore 条目存储在独立的 collection 中
- collection 命名格式：`story_{story_id}`
- 数据自动持久化，无需手动备份

向量库数据会随 Lore CRUD 操作自动更新，无需额外维护。

## 13. 基础安全加固，确认 SSH 密钥登录后再执行

确认你可以用密钥登录 `dreamweave` 用户后，再考虑禁用 root 密码登录。

先测试：

```powershell
ssh dreamweave@47.101.67.241
```

确认可登录后，在服务器执行：

```bash
cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak.$(date +%Y%m%d%H%M%S)
sed -i 's/^#\\?PasswordAuthentication .*/PasswordAuthentication no/' /etc/ssh/sshd_config
sed -i 's/^#\\?PermitRootLogin .*/PermitRootLogin prohibit-password/' /etc/ssh/sshd_config
sshd -t
systemctl reload ssh
```

不要在没验证密钥登录前执行这一步，避免把自己锁在服务器外。

## 14. 每日 PostgreSQL 备份

创建备份目录：

```bash
mkdir -p /opt/dreamweave/backups/postgres
chmod 700 /opt/dreamweave/backups/postgres
```

创建备份脚本：

```bash
cat > /opt/dreamweave/backup-postgres.sh <<'EOF'
#!/usr/bin/env bash
set -euo pipefail

source /opt/dreamweave/.env

BACKUP_DIR="/opt/dreamweave/backups/postgres"
DATE="$(date +%Y%m%d_%H%M%S)"
FILE="${BACKUP_DIR}/dreamweave_${DATE}.sql.gz"

pg_dump "$DATABASE_URL" | gzip > "$FILE"
find "$BACKUP_DIR" -type f -name "dreamweave_*.sql.gz" -mtime +14 -delete
EOF

chmod +x /opt/dreamweave/backup-postgres.sh
```

加入 cron：

```bash
(crontab -l 2>/dev/null; echo "20 3 * * * /opt/dreamweave/backup-postgres.sh") | crontab -
crontab -l
```

手动测试：

```bash
/opt/dreamweave/backup-postgres.sh
ls -lh /opt/dreamweave/backups/postgres
```

后续建议把备份同步到服务器外部位置，例如 OSS。
