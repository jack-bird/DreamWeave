# 织梦纪 DreamWeave 项目进度

更新日期：2026-05-26

## 1. 当前阶段

当前项目已经跑通第一版核心线上闭环，进入第一版数据持久化接入和验证阶段。

整体目标仍然是先跑通：

```text
用户输入 → 服务器转发 → 本地 Worker → Ollama 生成 → 返回剧情
```

目前已经完成本地 `Ollama + Python Worker` 基础能力、线上 `apps/server` 部署、第一版 H5 页面、数据库设计和初始化 SQL，以及 `apps/server` 的第一版 PostgreSQL 持久化代码。

公网最小闭环已经跑通：

```text
公网 H5 页面输入
        ↓
线上 Server HTTP 接收请求
        ↓
线上 Server 通过 WebSocket 下发 AI 任务
        ↓
本地 Worker 调用 Ollama
        ↓
Worker 返回 AI 结果
        ↓
线上 Server 返回给 H5 显示
```

当前线上 server 已通过 PM2 启动，`http://47.101.67.241/health` 已验证正常，本地 Worker 已连接线上 `/ws/worker`，`worker_count = 1`。

线上阿里云 Ubuntu 24.04 服务器初始化、代码部署、migration、PM2 启动和公网健康检查已经完成：

```text
基础系统环境 → 部署用户 → 防火墙 → Node.js/PM2 → PostgreSQL → Redis → Nginx → /opt/dreamweave → git clone → migration → PM2 → /health
```

当前本地仓库记录显示 `master` 与 `origin/master` 都指向提交 `d861b94 Add PostgreSQL persistence for story sessions`，该提交已包含本次 PostgreSQL 持久化代码。下一步是在服务器 `git pull`、`npm install --omit=dev`、`pm2 restart dreamweave-server --update-env` 后验证消息是否写入数据库。

## 6. 当前关键决策

### 6.1 模型选择

模型不应写死在 Worker 中。

当前规则：

```text
任务 model 字段优先
        ↓
Worker 默认模型兜底
        ↓
模型不存在则返回 MODEL_NOT_FOUND
```

这意味着后续可以按用户、故事、会话或前端选择动态切换模型。

### 6.2 第一版任务队列

第一版可以先用内存 pending task 跑通闭环。

Redis / BullMQ 建议在核心链路跑通后接入。

### 6.3 第一版数据库

数据库不必阻塞 WebSocket 联调。

建议先跑通：

```text
HTTP 请求 → WebSocket Worker → Ollama → HTTP 返回
```

再补：

```text
保存用户输入和 AI 回复
```

### 6.4 Prompt 策略

第一版 Prompt 先追求稳定输出，不追求复杂 Agent。

当前重点：

- 输出干净。
- 不替用户做决定。
- 保持剧情推进。
- 方便前端直接展示。

## 8. 当前风险和挑战

### 8.1 模型输出仍需持续调优

Prompt 已经明显收紧，但不同模型仍可能出现风格、长度、动作边界不稳定的问题。

后续需要在真实交互中持续调整 Prompt、生成参数和后处理规则。

### 8.2 任务可靠性尚未完成

当前还没有持久任务队列。

如果服务器或 Worker 中途断开，任务恢复、重试和状态追踪还需要后续实现。

### 8.3 多用户系统复杂度

当前第一版主要是单用户体验，后续扩展到多用户时需要考虑：

- 用户身份认证和授权
- 数据隔离和隐私保护
- 并发访问和性能优化
- 成本控制和资源管理

建议在完成核心功能后再逐步引入多用户支持。
