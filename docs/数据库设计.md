# 织梦纪 DreamWeave 数据库设计

版本：v0.1  
更新日期：2026-05-25

## 1. 设计目标

第一版数据库只服务最小互动闭环：

```text
保存用户
保存小说项目
保存会话
保存用户输入
保存 AI 回复
保存 AI 任务状态
```

暂不设计复杂长期记忆、推荐系统、支付、多人在线和复杂 Agent 状态。

---

## 2. 技术选择

第一版使用：

```text
PostgreSQL
```

原因：

- 关系结构清晰，适合用户、故事、会话、消息等核心数据。
- `jsonb` 方便保存 AI 上下文、生成参数和任务元数据。
- 后续可以平滑接入全文搜索、向量扩展或任务审计。

---

## 3. ID 策略

第一版使用应用层生成的 `text` ID。

示例：

```text
user_local
story_local
session_local
msg_xxx
task_xxx
```

原因：

- 当前 server / Worker 已经使用字符串 ID。
- 本地开发、调试和日志排查更直观。
- 后续如果需要 UUID，可以单独做迁移。

---

## 4. 核心实体关系

```text
users 1 ── n stories
users 1 ── n sessions
stories 1 ── n sessions
sessions 1 ── n messages
sessions 1 ── n ai_tasks
ai_tasks 1 ── 0..n messages
```

解释：

- 一个用户可以创建多个小说项目。
- 一个小说项目可以有多个会话。
- 一个会话包含多条消息。
- 每次 AI 生成会创建一条 AI 任务记录。
- AI 任务成功后通常会生成一条 `assistant` 消息。

## 5. 表结构

### 5.1 users

用户表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `text` | 用户 ID，主键 |
| `email` | `text` | 邮箱，可为空，唯一 |
| `nickname` | `text` | 昵称 |
| `created_at` | `timestamptz` | 创建时间 |
| `updated_at` | `timestamptz` | 更新时间 |

第一版本地开发可以固定使用：

```text
user_local
```

### 5.2 stories

小说项目表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `text` | 小说项目 ID，主键 |
| `user_id` | `text` | 所属用户 |
| `title` | `text` | 小说标题 |
| `world_setting` | `text` | 世界观设定 |
| `character_setting` | `text` | 主角或用户角色设定 |
| `default_model` | `text` | 默认模型 |
| `generation_options` | `jsonb` | 默认生成参数 |
| `created_at` | `timestamptz` | 创建时间 |
| `updated_at` | `timestamptz` | 更新时间 |

### 5.3 sessions

会话表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `text` | 会话 ID，主键 |
| `user_id` | `text` | 所属用户 |
| `story_id` | `text` | 所属小说项目 |
| `title` | `text` | 会话标题 |
| `status` | `text` | 会话状态 |
| `created_at` | `timestamptz` | 创建时间 |
| `updated_at` | `timestamptz` | 更新时间 |

`status` 可选值：

```text
active
archived
deleted
```

### 5.4 messages

消息表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `text` | 消息 ID，主键 |
| `session_id` | `text` | 所属会话 |
| `role` | `text` | 消息角色 |
| `content` | `text` | 消息内容 |
| `model` | `text` | AI 回复使用的模型，用户消息可为空 |
| `task_id` | `text` | 关联 AI 任务，可为空 |
| `metadata` | `jsonb` | 扩展信息 |
| `created_at` | `timestamptz` | 创建时间 |

`role` 可选值：

```text
user
assistant
system
```

第一版读取历史记录时，按：

```sql
session_id + created_at + id
```

排序。

### 5.5 ai_tasks

AI 任务表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `text` | 任务 ID，主键 |
| `task_type` | `text` | 任务类型 |
| `status` | `text` | 任务状态 |
| `user_id` | `text` | 用户 ID |
| `story_id` | `text` | 小说项目 ID |
| `session_id` | `text` | 会话 ID |
| `worker_id` | `text` | 执行任务的 Worker |
| `request_id` | `text` | 协议请求 ID |
| `model` | `text` | 实际使用模型 |
| `input` | `text` | 用户输入 |
| `context` | `jsonb` | 下发给 Worker 的上下文 |
| `generation_options` | `jsonb` | 生成参数 |
| `output` | `text` | AI 输出内容 |
| `error_code` | `text` | 错误码 |
| `error_message` | `text` | 错误信息 |
| `retryable` | `boolean` | 是否可重试 |
| `timeout_ms` | `integer` | 任务超时时间 |
| `duration_ms` | `integer` | 执行耗时 |
| `created_at` | `timestamptz` | 创建时间 |
| `started_at` | `timestamptz` | 开始时间 |
| `completed_at` | `timestamptz` | 完成时间 |
| `updated_at` | `timestamptz` | 更新时间 |

`task_type` 第一版只要求：

```text
story_continue
```

`status` 可选值：

```text
created
sent_to_worker
running
success
error
timeout
cancelled
```

## 6. 第一版写入流程

### 6.1 用户发送消息

```text
1. 确保 user / story / session 存在
2. 插入 user message
3. 创建 ai_task，状态为 created
4. Server 发送任务给 Worker，状态更新为 sent_to_worker
5. Worker 返回结果
6. 更新 ai_task 为 success 或 error
7. 如果成功，插入 assistant message
8. 更新 session.updated_at
```

### 6.2 页面加载历史

```text
1. 查询当前用户的 stories
2. 查询 story 下的 sessions
3. 查询 session 下的 messages
4. 按 created_at 升序显示
```

## 7. 第一版 API 对数据库的影响

当前接口：

```text
POST /api/story/continue
```

接入数据库后需要：

- 请求开始时保存用户消息。
- 创建 AI 任务记录。
- AI 成功后保存 assistant 消息。
- AI 失败后保存任务错误。
- 返回给前端时附带 `session_id`、`story_id`、`task_id`。

后续需要新增：

```text
GET /api/stories
GET /api/stories/:story_id/sessions
GET /api/sessions/:session_id/messages
POST /api/stories
POST /api/sessions
```

---

## 8. 索引策略

第一版需要的索引：

- `stories(user_id, created_at)`
- `sessions(user_id, updated_at)`
- `sessions(story_id, updated_at)`
- `messages(session_id, created_at, id)`
- `ai_tasks(session_id, created_at)`
- `ai_tasks(status, created_at)`
- `ai_tasks(worker_id, created_at)`

---

## 9. 第一版暂不做

暂不做：

- 多人协作权限。
- 复杂故事版本管理。
- 长期记忆向量库。
- 消息全文搜索。
- AI 任务自动重试队列。
- 多 Worker 调度历史统计。
- 用户付费和用量计费。
