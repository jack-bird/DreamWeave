# 织梦纪 DreamWeave 初期执行框架

## 1. 项目定位

织梦纪是一个基于 AI 小说框架的互动小说应用。

用户通过手机端输入文字，与小说剧情、角色和世界观进行交互。AI 根据当前剧情、角色设定、历史上下文和用户输入，生成后续小说内容。

第一阶段的重点不是功能完整，而是先完成一个稳定、可保存、可扩展的最小互动闭环。

---

## 2. 第一版目标

第一版只完成核心闭环：

```text
用户在手机端输入内容
        ↓
线上服务器接收请求
        ↓
服务器转发给本地 AI Worker
        ↓
本地 Python Worker 调用 Ollama 模型
        ↓
Ollama 返回 AI 小说内容
        ↓
服务器返回给手机端显示
```

第一版完成后，用户应该可以在手机网页中输入一句话，并获得一段符合小说设定的 AI 互动剧情。

---

## 3. 初期系统架构

```text
手机端 Web / H5
        ↓
线上 Linux 服务器
        ↓ WebSocket 长连接
本地 Python AI Worker
        ↓
Ollama 本地模型
```

核心原则：

- 本地电脑不暴露公网。
- 本地 Worker 主动连接线上服务器。
- 服务器只负责转发、排队、保存和返回结果。
- AI 推理运行在本地 Ollama。

## 5. 模块职责

### 5.1 手机端 Web / H5

负责：

- 用户登录
- 选择小说项目
- 输入互动内容
- 显示 AI 回复
- 显示剧情记录
- 显示角色信息

第一版先做 H5 页面，不急着做原生 App。

### 5.2 线上服务器

负责：

- 用户请求入口
- 会话管理
- 数据库存储
- 请求转发
- 任务队列
- WebSocket 连接管理
- 返回 AI 结果给前端

建议技术栈：

```text
Node.js / NestJS
PostgreSQL
Redis
WebSocket
```

### 5.3 本地 AI Worker

负责：

- 主动连接线上服务器
- 接收 AI 任务
- 组装 Prompt
- 调用 Ollama
- 流式返回生成内容
- 处理失败重试
- 断线自动重连

建议技术栈：

```text
Python
websockets
httpx
FastAPI 可选
```

第一版 Worker 通过 Ollama 本地 HTTP API 调用模型，不依赖 Python `ollama` 包。

### 5.4 Ollama 本地模型

负责：

- 小说续写
- 角色扮演
- 剧情推进
- 对话生成
- 内容总结

## 6. 第一版核心流程

```text
1. 用户打开手机网页
2. 用户输入一句话
3. 前端发送请求到服务器
4. 服务器创建 AI 任务
5. 服务器通过 WebSocket 发送任务给本地 Worker
6. Worker 组装 Prompt 并调用 Ollama
7. Ollama 生成小说回复
8. Worker 返回结果给服务器
9. 服务器保存消息记录
10. 服务器把结果返回给前端
11. 前端显示 AI 回复
```

## 7. 第一版数据结构

### 7.1 用户输入请求

```json
{
  "user_id": "user_001",
  "session_id": "session_001",
  "story_id": "story_001",
  "message": "我推开了古堡的大门。"
}
```

### 7.2 AI 任务结构

```json
{
  "task_id": "task_001",
  "task_type": "story_continue",
  "user_id": "user_001",
  "session_id": "session_001",
  "story_id": "story_001",
  "model": "qwen3:14b",
  "timeout_ms": 180000,
  "input": "我推开了古堡的大门。",
  "generation_options": {
    "num_predict": 220,
    "temperature": 0.66,
    "top_p": 0.85,
    "think": false
  },
  "context": {
    "story_title": "黑夜古堡",
    "world_setting": "中世纪奇幻世界",
    "character_setting": "用户是失忆的贵族继承人",
    "recent_messages": []
  }
}
```

说明：

- `model` 为可选字段。任务指定模型时优先使用任务模型，否则使用 Worker 默认模型。
- `generation_options` 为可选字段，用于覆盖单次生成参数。
- 如果任务指定的模型在本地 Ollama 中不存在，Worker 应返回 `MODEL_NOT_FOUND`。

### 7.3 AI 返回结构

```json
{
  "task_id": "task_001",
  "status": "success",
  "model": "qwen3:14b",
  "content": "沉重的木门缓缓打开，腐朽的冷风从黑暗深处涌出……"
}
```

### 7.4 AI 错误返回结构

```json
{
  "task_id": "task_001",
  "status": "error",
  "model": "qwen3:14b",
  "error_code": "OLLAMA_TIMEOUT",
  "message": "Ollama 生成超时"
}
```

## 8. Prompt 当前模板

```text
你是织梦纪的互动小说续写引擎。

你的任务是根据用户输入、小说设定、角色设定和最近剧情，续写下一段沉浸式小说正文。

严格输出规则：
1. 只输出小说正文，不要输出解释、标题、角色名、时间戳、系统提示或 Markdown。
2. 不要输出“我该怎么办”“请选择”“请直接输入”等界面提示。
3. 不要输出选项列表，不要使用项目符号或编号。
4. 不要替用户做重大决定，不要代替用户说出台词。
5. 用户输入是主角最后一个已经完成的动作；续写中不要让主角再主动执行新的身体动作或选择。
6. 禁止描写主角继续前进、后退、逃跑、攻击、开门、选择道路、说话、拿取物品或做决定。
7. 允许推动外部事件，例如环境变化、声音、气味、危险逼近、NPC 出现、记忆闪回和角色感受。
8. 承认用户输入的动作已经发生，重点描写该动作造成的后果。
9. 保持角色、世界观和氛围一致。
10. 输出 2 到 3 个自然段，总长度控制在 180 到 360 个中文字符。
11. 结尾必须是完整句子，可以留下悬念或可回应空间，但仍然必须是小说正文。

小说标题：
{{story_title}}

世界观设定：
{{world_setting}}

角色设定：
{{character_setting}}

最近剧情：
{{recent_messages}}

用户输入：
{{user_input}}

现在开始续写，只输出小说正文：
```

## 9. 初期开发顺序

### 阶段 1：本地模型跑通

目标：

- 安装 Ollama
- 下载并验证小说生成模型
- Python 可以调用 Ollama
- 输入一句话，返回小说内容

### 阶段 2：本地 Worker 跑通

目标：

- Python Worker 可以启动
- Worker 主动连接服务器 WebSocket
- Worker 可以接收任务
- Worker 可以调用 Ollama
- Worker 可以返回 AI 结果

### 阶段 3：服务器转发跑通

目标：

- 服务器接收前端请求
- 服务器创建任务 ID
- 服务器发送任务给本地 Worker
- 服务器接收 Worker 返回
- 服务器返回结果给前端

### 阶段 4：前端页面跑通

目标：

- 手机网页可以输入文字
- 页面可以显示 AI 回复
- 页面可以显示历史对话

### 阶段 5：数据库接入

目标：

- 保存用户
- 保存小说项目
- 保存会话
- 保存用户输入
- 保存 AI 回复

## 11. 必须注意的问题

### 11.1 本地电脑不要暴露公网

本地 Worker 必须主动连接服务器。不要让服务器直接访问本地电脑。

### 11.2 必须有断线重连

Worker 需要支持：

- 自动重连
- 心跳检测
- 任务超时
- 错误返回

### 11.3 必须限制并发

初期建议：

```text
同时最多处理 1-3 个 AI 任务
```

### 11.4 必须有任务队列

服务器不要直接把大量请求打到本地模型。

后续建议接入：

```text
Redis Queue / BullMQ
```

## 12. 第一版暂不实现的内容

第一版暂时不做：

- 支付系统
- 复杂多模型调度和自动路由
- 图片生成
- 复杂 Agent
- 长期记忆
- 语音
- App 上架
- 多人在线
- 推荐系统

先跑通核心闭环，再逐步扩展。

## 13. 第一版验收标准

当以下流程成功，就算第一版完成：

```text
用户在手机网页输入：
“我走进森林深处。”

系统返回：
一段符合小说设定的 AI 互动剧情。
```

同时满足：

- 前端能显示 AI 回复。
- 服务器能转发任务。
- 本地 Worker 能调用 Ollama。
- 数据库能保存记录。
- Worker 断线后能自动重连。

---

## 14. 项目核心原则

第一阶段只追求：

```text
能跑通
能保存
能互动
能扩展
```

不要一开始追求完美。

织梦纪第一版的核心不是功能多，而是完成这个最小闭环：

```text
用户输入 → AI 生成 → 剧情返回
```
