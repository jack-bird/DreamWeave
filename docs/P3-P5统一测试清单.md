# DreamWeave P3-P5 统一测试清单

更新日期：2026-05-30

## 1. 测试目标

本清单用于服务器 P3/P4 部署完成、本地 Worker 向量库依赖安装完成后，统一验证以下链路：

```text
作者后台作品管理
        -> Lore CRUD
        -> Server 写入 PostgreSQL
        -> Server 通过 WebSocket 通知本地 Worker
        -> Worker 写入本地 Chroma
        -> 玩家大厅和开始游玩链路继续可用
```

## 5. P5 RAG 索引初步测试

当前 RAG 索引由 Server 通过 WebSocket 发 `rag.task` 给本地 Worker，Worker 负责写入本地 Chroma。

测试步骤：

1. 保持本地 Worker 正在运行。
2. 在作者后台新增一条 Lore。
3. 观察本地 Worker 窗口是否出现 RAG / Chroma / indexing 相关日志。
4. 查看本地项目目录是否生成 `chroma_db`。

本地确认：

```powershell
cd E:\ai_home\AI_Projects\DreamWeave
dir .\chroma_db
```

通过标准：

- 新增 Lore 不报错。
- 本地 Worker 不崩溃。
- 本地出现 `chroma_db` 目录。
- 后续编辑 / 删除 Lore 时 Worker 仍能继续运行。

注意：Chroma 是本地 Worker 的语义索引，不是主数据库。即使 Chroma 索引失败，PostgreSQL 中的 Lore 原文仍应保存成功。

## 6. 玩家生成链路回归

测试步骤：

1. 进入玩家大厅。
2. 打开刚发布的作品。
3. 开始游玩或进入已有 session。
4. 输入一条玩家行动。
5. 等待 AI 回复。

服务器复查：


通过标准：

- `messages` 写入用户输入和 AI 回复。
- `ai_tasks` 最近任务为 `success`。
- `story_states.version` 随生成更新。

## 7. 常见问题定位

### 7.1 作者后台保存失败

先查 Server 日志：

```bash
pm2 logs dreamweave-server
```

再确认数据库表：

```bash
source /opt/dreamweave/.env
psql "$DATABASE_URL" -c "\d stories"
psql "$DATABASE_URL" -c "\d lore_entries"
```

### 7.2 Worker 未连接

服务器检查：

```bash
curl http://127.0.0.1:3000/health
```

如果 `worker_count = 0`，重启本地 Worker：

```powershell
cd E:\ai_home\AI_Projects\DreamWeave
.\start-online-worker.bat
```

### 7.3 RAG 索引失败

本地确认依赖：

```powershell
cd E:\ai_home\AI_Projects\DreamWeave
. E:\ai_home\AI_Projects\llm_env\Scripts\Activate.ps1
python -c "import chromadb; import sentence_transformers; print('RAG dependencies OK')"
```

如果依赖正常，查看本地 Worker 窗口日志。

### 7.4 AI 生成失败

本地确认 Ollama：

```powershell
ollama list
python .\apps\local-ai-worker\main.py health
```

服务器确认最近任务：

```bash
psql "$DATABASE_URL" -c "SELECT id, status, error_code, error_message, created_at FROM ai_tasks ORDER BY created_at DESC LIMIT 10;"
```
